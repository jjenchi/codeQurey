# CodeQuery 部署清單

> 最後更新：2026-06-13
> 本文件記錄完整的部署過程，讓 Jackie 或其他人能照著操作

## 第一階段：申請 AWS 帳號

### 1.1 前往 AWS 官網

1. 開啟瀏覽器，前往 https://aws.amazon.com/
2. 點擊右上角「Create an AWS Account」（建立 AWS 帳號）

### 1.2 填寫帳號資料

1. **Email**：輸入用來管理 AWS 的 email
2. **AWS account name**：取一個帳號名稱（例：`codequery` 或公司名稱）
3. 點擊「Verify email address」→ 到信箱收驗證碼 → 輸入驗證碼

### 1.3 設定密碼

1. 設定 Root user password（至少 8 碼，含大小寫+數字+符號）
2. 這組是最高權限帳號，請妥善保管

### 1.4 聯絡資訊

1. Account type 選「Personal」或「Business」皆可
2. 填寫姓名、電話、地址
3. 勾選同意條款

### 1.5 付款資訊

1. 輸入信用卡號（Visa / Mastercard / JCB 都可以）
2. AWS 會先扣 $1 USD 驗證（之後退回）
3. 新帳號有 12 個月免費額度（Free Tier）

### 1.6 身份驗證

1. 選擇「Text message (SMS)」
2. 輸入手機號碼（台灣：+886）
3. 收到驗證碼 → 輸入

### 1.7 選擇 Support Plan

1. 選「Basic support - Free」（免費方案）
2. 點擊「Complete sign up」

### 1.8 完成

- 帳號建立完成，需要幾分鐘啟用
- 收到確認 email 後就可以登入 AWS Console

## 第二階段：建立 EC2 主機

### 2.1 進入 EC2 服務

1. 登入 AWS Console
2. 右上角區域選擇 **亞太區域（東京）ap-northeast-1**
3. 搜尋列輸入「EC2」→ 點擊進入

### 2.2 啟動執行個體

1. 點擊「啟動執行個體」
2. **名稱**：輸入 `CodeQuery`
3. **AMI**：選擇 `Amazon Linux 2023`（預設，Free Tier 標示）
4. **執行個體類型**：選擇 `t3.micro`（Free Tier 免費）
   - ⚠️ 正式使用建議升級至 `t3.small` 或 `t3.medium`（效能較好）

### 2.3 金鑰對

1. 點擊「建立新的金鑰對」
2. **名稱**：`codequery-key`
3. **類型**：RSA
4. **格式**：`.pem`
5. 點擊「建立金鑰對」→ 瀏覽器會自動下載 `codequery-key.pem`
6. ⚠️ **這個檔案非常重要**，是 SSH 連線到主機的唯一方式，請妥善保管
7. ⚠️ **不要把 .pem 檔上傳到 GitHub**（已加入 .gitignore）

### 2.4 防火牆（安全群組）

1. 選擇「建立安全群組」
2. ✅ 勾選「允許 SSH 流量」（來源：0.0.0.0/0）
3. ✅ 勾選「允許來自網際網路的 HTTP 流量」
4. ❌ HTTPS 不用勾（目前不需要）

### 2.5 儲存空間

1. 使用預設：8 GB gp3
2. 不需要修改

### 2.6 啟動

1. 點擊「啟動執行個體」
2. 看到綠色「成功」訊息 → 記錄 Instance ID
3. **Instance ID**：記下來（例如 `i-0a1ba37ff66b0cc61`）

### 2.7 取得公有 IP

1. 點擊左側選單「執行個體」
2. 找到 CodeQuery 執行個體
3. 記下「公有 IPv4 位址」欄位 → 這就是之後連線和存取用的 IP
4. **公有 IP**：記下來，之後所有步驟都會用到

## 第三階段：安裝軟體與部署程式

### 3.1 SSH 連線到主機

```bash
# 設定金鑰權限（只需做一次）
chmod 400 codequery-key.pem

# 連線
ssh -i codequery-key.pem ec2-user@<你的EC2公有IP>
```

### 3.2 安裝 Node.js、Git、PM2

```bash
# 安裝 Node.js 和 Git
sudo dnf install -y nodejs git

# 安裝 PM2（程序管理，讓後端常駐運行）
sudo npm install -g pm2
```

### 3.3 上傳程式碼

從本地電腦執行（不是在 EC2 上）：

```bash
# 建立資料夾
ssh -i codequery-key.pem ec2-user@<你的EC2公有IP> "mkdir -p ~/codequery/backend ~/codequery/frontend ~/codequery/repos"

# 上傳後端
scp -i codequery-key.pem backend/server.mjs backend/package.json backend/init-admin.mjs backend/.env ec2-user@<你的EC2公有IP>:~/codequery/backend/

# 上傳前端
scp -i codequery-key.pem frontend/index.html frontend/admin.html ec2-user@<你的EC2公有IP>:~/codequery/frontend/

# 上傳設定檔（如果有的話）
scp -i codequery-key.pem projects.json users.json ec2-user@<你的EC2公有IP>:~/codequery/
```

### 3.4 安裝 npm 依賴

```bash
# SSH 進主機後
cd ~/codequery/backend
npm install
```

### 3.5 初始化管理員帳號

```bash
cd ~/codequery/backend
node init-admin.mjs <帳號> <密碼> <email>
```

### 3.6 啟動後端

```bash
# 用 PM2 啟動
cd ~/codequery/backend
pm2 start server.mjs --name codequery

# 設定開機自動啟動
pm2 startup
# 會印出一行 sudo 指令，複製貼上執行
pm2 save
```

### 3.7 驗證

- 查詢頁面：`http://<你的EC2公有IP>:5001/index.html`
- 管理頁面：`http://<你的EC2公有IP>:5001/admin.html`
- 健康檢查：`http://<你的EC2公有IP>:5001/api/health`

### 3.8 常用 PM2 指令

```bash
pm2 status          # 查看狀態
pm2 logs codequery  # 查看即時 log
pm2 restart codequery  # 重啟
pm2 stop codequery     # 停止
```

### 3.9 更新程式碼

從本地電腦執行：

```bash
# 重新上傳修改過的檔案
scp -i codequery-key.pem backend/server.mjs ec2-user@<你的EC2公有IP>:~/codequery/backend/

# SSH 進主機重啟
ssh -i codequery-key.pem ec2-user@<你的EC2公有IP> "pm2 restart codequery"
```

## AI Model 設定

### 自動偵測機制

Server 啟動時會自動執行：

1. 呼叫 `Cursor.models.list()` 查詢所有可用模型
2. 篩出 `claude-opus-*` 系列，取版本最新的
3. 啟動 log 印出「AI 模型：claude-opus-X-X」

**手動覆蓋**：在 `.env` 加一行 `AI_MODEL=claude-opus-4-7` 可鎖定特定版本。

**模型出錯時**：
- 啟動階段：如果 `.env` 指定的模型不在可用清單 → log 警告 + 自動切最新 Opus
- 查詢階段：如果模型回應異常 → log 印出「目前使用的模型可能已無法使用」+ 建議重啟

### ⚠️ 重要限制：僅支援 Claude 系列模型

目前程式的回應解析是按照 Claude 的 event stream 格式設計的。**不要切到非 Claude 的模型**（如 GPT、Gemini、Grok），否則回應會解析失敗。

如果未來需要支援其他模型，需要修改 `server.mjs` 中 `run.stream()` 的事件解析邏輯。

### Claude 系列 Event Stream 格式

- 事件類型：`status`、`thinking`、`tool_call`、`assistant`
- 答案在 `event.message.content[].text`（`assistant` 事件）
- `thinking` 是思考過程，不顯示給使用者

### 可用模型參考

完整清單可用 `Cursor.models.list()` 即時查詢。以下為參考（會隨版本更新）：

| Model ID | 說明 |
|----------|------|
| `claude-opus-4-8` | Claude Opus 4.8（最新） |
| `claude-opus-4-7` | Claude Opus 4.7 |
| `claude-sonnet-4-6` | Claude Sonnet 4.6（較便宜） |

| 項目 | 費用 |
|------|------|

## 需要的帳號 / Key 總覽

| 項目 | 用途 | 誰提供 |
|------|------|--------|
| AWS 帳號 | EC2 主機 + SES 寄信 | Charles 或 Jackie |
| Cursor API Key | 呼叫 Cursor SDK | cursor.com/settings |
| GitHub PAT | clone private repo | 各開發人員自己建 |

### GitHub Token 建立步驟（給開發人員）

1. GitHub → Settings → Developer settings → Personal access tokens → Fine-grained tokens
2. Generate new token
3. Repository access：Only select repositories → 選要加入的 repo
4. Permissions → Contents：Read-only
5. 複製 token，到管理頁面新增專案時貼上
