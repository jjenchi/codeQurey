# CodeQuery 架構設計

> 讓非技術人員（客服/業務）用自然語言查詢程式碼資訊的 Web App
> 最後更新：2026-06-13（v3.0：登入驗證 + 安全防護 + 角色分層）

## 使用情境

### 情境一：業務談案子
客戶問功能做到哪、未來能做什麼、完成度如何。業務需要即時查詢回答。

### 情境二：程式出錯
現場人員需要初步判斷問題方向，先回覆客戶安心，後續再由工程師處理。

## 使用者角色

| 角色 | 做什麼 |
|------|--------|
| 開發人員 | 維護程式碼，push 到 GitHub；透過管理頁面新增/管理專案 |
| 客服/業務 | 透過查詢頁面用自然語言提問，取得非技術語言的回答 |

## 架構演進紀錄

| 版本 | 架構 | 結果 |
|------|------|------|
| v0 | Claude API + 自寫 Agent（grep/find/cat 工具）| 能用，但 55 秒 / 5 次 API 呼叫，太慢 |
| v1 | README 索引 → 直接塞 .cpp → 一次 API 呼叫 | 快（5 秒），但搜尋規則寫不完善，泛關鍵字爆 token |
| v1.5 | v1 + Agent fallback（README 找不到時）| 混合模式，但關鍵字提取仍是瓶頸 |
| v2 | Cursor SDK（不用自己寫搜尋規則）| Cursor agent 引擎全包，單一專案 |
| v2.1 | v2 + 多專案 + 管理頁面 | 支援多開發者多專案，兩層 repo 結構 |
| **v3.0** | **v2.1 + 登入驗證 + Rate Limit + AI 安全分類器 + 三層角色（本版）** | **查詢頁面需登入、每分鐘 10 次限制、Regex + AI 雙層程式碼防護、admin/editor/viewer 角色分層** |

## v3.0 安全機制

### 三層角色

| 角色 | 查詢頁面 | 管理專案 | 管理帳號 | 查看 log | 清除 log |
|------|---------|---------|---------|---------|---------|
| admin | O | O | O | O | O |
| editor | O | O | X | O | X |
| viewer | O | X | X | X | X |

### 查詢頁面登入

- 查詢頁面（index.html）需要帳號密碼登入才能使用
- 登入成功後 JWT token 存在 sessionStorage
- Token 過期自動導回登入畫面

### Rate Limiting

- 每個使用者每分鐘最多 10 次查詢
- 超過限制回傳 429 錯誤

### 程式碼防護（雙層）

| 層級 | 機制 | 攔截對象 |
|------|------|---------|
| 第一層 | Regex 正則比對 | 明顯的程式碼請求、prompt injection、jailbreak |
| 第二層 | AI 安全分類器（composer-2.5） | 多語言繞過、同義詞繞過、間接提取 |

### 子資料夾隔離

- 選擇子資料夾後，Agent 被限制只能搜尋該子資料夾內的檔案
- 找不到時回答「在此子專案中沒有找到相關資訊」，不會去其他資料夾找

## v2.1 系統架構

```
客服的瀏覽器（手機/電腦）
  │
  │  選擇專案 → 傳送「問題」→ 接收「答案」
  │
  ↓
前端
  ├── 查詢頁面（index.html）── 專案下拉選單 + 聊天介面
  └── 管理頁面（admin.html）── 新增/編輯/刪除專案
  │
  ↓
Node.js 後端（Express）
  ├── /api/query            ← 接收問題，呼叫 Cursor SDK Agent
  ├── /api/projects         ← 專案 CRUD（讀寫 projects.json）
  ├── /api/health           ← 健康檢查
  │
  ├── Cursor SDK Agent
  │     ├── 自動做 codebase indexing
  │     ├── 自動做 semantic search + grep
  │     ├── 自動讀取相關檔案
  │     └── 用 AI 組合答案
  │
  └── System Prompt 控制回答風格（非技術語言）
  │
  ↓
repos/（clone 在後端同一台機器）
  ├── jjenchi/CNC/
  ├── jjenchi/CNC-v2/
  └── another-dev/laser-ctrl/
```

## 設定結構

### 全域設定（`.env`）

| 變數 | 用途 | 必填 |
|------|------|------|
| `CURSOR_API_KEY` | Cursor SDK API Key | ✅ |
| `PORT` | 後端 port（預設 5001） | ✅ |
| `JWT_SECRET` | JWT 登入驗證簽章密鑰 | ✅ |
| `AI_MODEL` | 指定 AI 模型（留空則自動偵測最新 Claude Opus） | 可選 |

### 專案設定（`projects.json`）

每筆專案獨立管理，透過管理頁面操作：

```json
[
  {
    "developer": "jjenchi",
    "name": "CNC",
    "display_name": "CNC 控制系統",
    "git_url": "https://github.com/jjenchi/CNC.git",
    "git_token": "ghp_..."
  }
]
```

- `developer` + `name` 組成唯一識別，也是 repo 資料夾路徑
- `git_token` 每個專案獨立，支援不同開發者帳號
- `display_name` 顯示在前端下拉選單

## API 端點

### 公開端點（不需登入）

| 方法 | 路徑 | 功能 |
|------|------|------|
| POST | `/api/auth/login` | 登入，回傳 JWT token |
| POST | `/api/auth/forgot` | 忘記密碼（寄驗證碼） |
| POST | `/api/auth/reset` | 重設密碼（驗證碼） |
| GET | `/api/health` | 健康檢查 |

### 查詢端點（需登入）

| 方法 | 路徑 | 功能 |
|------|------|------|
| GET | `/api/projects` | 列出所有專案（隱藏 token，供查詢頁下拉選單） |
| POST | `/api/query` | 查詢（需帶 developer + name + question，Rate Limit 10 次/分） |

### 管理端點（需登入，部分需 Admin）

| 方法 | 路徑 | 需 Admin | 功能 |
|------|------|----------|------|
| GET | `/api/admin/projects` | ❌ | 專案列表（含 token） |
| GET | `/api/admin/projects/:dev/:name` | ❌ | 單一專案詳情 |
| POST | `/api/admin/projects` | ❌ | 新增專案 |
| PUT | `/api/admin/projects/:dev/:name` | ❌ | 更新專案 |
| DELETE | `/api/admin/projects/:dev/:name` | ❌ | 刪除專案 |
| GET | `/api/admin/users` | ✅ | 帳號列表 |
| POST | `/api/admin/users` | ✅ | 新增帳號 |
| DELETE | `/api/admin/users/:username` | ✅ | 刪除帳號 |
| PUT | `/api/admin/users/:username/password` | ✅ | 重設密碼 |
| GET | `/api/admin/logs` | ❌ | 操作紀錄（最近 100 筆） |
| DELETE | `/api/admin/logs` | ✅ | 清除操作紀錄 |

## 核心改變：不再自己寫搜尋規則

| | v1（自己寫） | v2/v2.1（Cursor SDK） |
|--|------------|----------------|
| 搜尋邏輯 | 自己寫關鍵字提取 + README 比對 | Cursor 的 semantic search + grep |
| 中文處理 | 自己做 bigram 切分 + stop words | Cursor AI 直接理解語意 |
| 追問處理 | 自己管 session + 歷史關鍵字合併 | Cursor agent 自動理解上下文 |
| 找不到時 | 自己寫 fallback 邏輯 | Cursor agent 自己決定下一步 |
| Token 控制 | 自己設上限、截斷 | Cursor agent 自己管理 |

## 部署方案

### 方案 A：開發測試（本機 Mac）

```
Mac
  ├── Node.js 後端（localhost:5001）
  ├── Cursor SDK（local 模式）
  ├── repos/（本機磁碟）
  └── 前端 HTML（瀏覽器直接開）
```

成本：Cursor Pro $20/月 + Cursor SDK token 費用

### 方案 B：正式部署（一台雲端主機）

```
AWS EC2（或任何 VPS）
  ├── Node.js 後端
  ├── Cursor SDK（local 模式）
  ├── repos/（磁碟上）
  └── 前端：S3 + CloudFront（或直接 EC2 serve）
```

| 服務 | 用途 | 月費估算 |
|------|------|----------|
| EC2 t3.medium | 後端 + Cursor SDK + repo | ~$30/月 |
| S3 + CloudFront | 前端靜態網頁 | < $1/月 |
| Cursor Pro | SDK 使用權 | $20/月 |
| Cursor SDK tokens | AI 查詢費用 | 依用量（$0.50/M input） |

### 方案 C：Serverless（未來考慮）

Lambda 不適合 Cursor SDK（需要持久的 codebase index），
除非用 Cursor cloud 模式（但 repo 要上傳到 Cursor 的 VM）。
等 Cursor SDK 更成熟後再評估。

## 設計決策紀錄

| 決策 | 原因 |
|------|------|
| Cursor SDK 取代自寫搜尋 | 搜尋規則寫不完善（中文 bigram 太粗、泛關鍵字爆 token、追問邏輯複雜），Cursor 的 semantic search + agent 已經解決這些問題 |
| Node.js 取代 Python | Cursor SDK 是 TypeScript，原生支援 |
| EC2 取代 Lambda | Cursor SDK 需要持久的 codebase index，serverless 冷啟動不適合 |
| Local 模式優先 | 不用把 private repo 上傳到 Cursor cloud，隱私較好 |
| 保留 System Prompt | 控制回答風格（非技術語言），跟 v1 一樣寫在後端 |
| 多專案獨立 token | 每個專案各自帶 git token，支援不同開發者/帳號的 repo |
| 兩層 repo 結構 | `repos/{開發者}/{專案名}/` 避免命名衝突，結構清晰 |
| 管理頁面取代改設定檔 | 即使技術人員操作，網頁介面比手動改 JSON 更不容易出錯 |
