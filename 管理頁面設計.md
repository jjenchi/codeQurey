# CodeQuery 管理頁面設計

> 最後更新：2026-06-13

## 概述

管理頁面（`frontend/admin.html`）供開發人員管理專案與帳號，與查詢頁面（`frontend/index.html`）完全分離。兩個頁面都需要帳號密碼登入。

## 角色與權限

| 功能 | Admin | Editor | Viewer |
|------|-------|--------|--------|
| 使用查詢頁面 | ✅ | ✅ | ✅ |
| 登入管理頁面 | ✅ | ✅ | ❌（API 回 403） |
| 查看專案列表 | ✅ | ✅ | ❌ |
| 新增專案 | ✅ | ✅ | ❌ |
| 編輯專案 | ✅ | ✅ | ❌ |
| 刪除專案 | ✅ | ✅ | ❌ |
| 查看操作紀錄 | ✅ | ✅ | ❌ |
| 清除操作紀錄 | ✅ | ❌ | ❌ |
| 查看帳號列表 | ✅ | ❌（Tab 隱藏） | ❌ |
| 新增帳號 | ✅ | ❌ | ❌ |
| 刪除帳號 | ✅ | ❌ | ❌ |
| 重設他人密碼 | ✅ | ❌ | ❌ |

## 帳號管理規則

- 新增帳號預設角色為 **Viewer**，可下拉選擇 Editor 或 Admin
- 新增帳號必須填寫：帳號、密碼、Email、角色（四欄全填才能儲存）
- **不可刪除自己的帳號**
- **最後一個 Admin 帳號不可刪除**（前端隱藏刪除按鈕 + 後端擋）
- 帳號資料存放：`users.json`（密碼以 bcrypt hash 儲存，不存明碼）

## 登入機制

- 登入成功後核發 JWT token，有效期 **24 小時**
- 查詢頁面 Token 存在 sessionStorage（關閉分頁即失效），管理頁面存在 localStorage（持久登入）
- 每次 API 請求帶在 Authorization header
- Token 過期後自動導回登入頁面

## 忘記密碼流程

```
使用者點「忘記密碼」
  → 輸入帳號
  → 後端產生 6 位數驗證碼，寄到帳號綁定的 Email
  → 使用者輸入驗證碼 + 新密碼
  → 驗證通過 → 密碼重設完成
```

- 驗證碼有效期：**10 分鐘**
- 寄信方式：SMTP（部署時設定 AWS SES 或其他 SMTP 服務）
- 無論帳號是否存在，API 都回傳成功（避免帳號探測）

## 操作紀錄

### 記錄的操作

| 操作 | 記錄內容 |
|------|---------|
| 登入 | 帳號 + 「登入成功」 |
| 新增專案 | 帳號 + developer/name |
| 更新專案 | 帳號 + developer/name |
| 刪除專案 | 帳號 + developer/name |
| 新增帳號 | 操作者帳號 + 被新增的帳號 |
| 刪除帳號 | 操作者帳號 + 被刪除的帳號 |
| 重設密碼 | 操作者帳號 + 被重設的帳號 |
| 透過驗證碼重設密碼 | 該帳號 + 「透過驗證碼重設」 |
| 清除紀錄 | 操作者帳號 + 「操作紀錄已被清除」 |

### 紀錄儲存機制

| 檔案 | 用途 | 可被清除 |
|------|------|---------|
| `admin-logs.json` | UI 顯示用（最近 100 筆） | ✅ Admin 可清除 |
| `admin-logs-archive.json` | 完整歷史備份 | ❌ 不可從 UI 清除 |

### 清除規則

- 只有 Admin 可以清除操作紀錄
- 清除前，舊紀錄自動備份到 `admin-logs-archive.json`（累積，不覆蓋）
- 清除後，主檔只留一筆「XXX 清除了操作紀錄」
- Archive 檔保留所有歷史，伺服器上永遠有完整紀錄

## API 端點與權限

| 方法 | 路徑 | 需登入 | 需 Admin | 說明 |
|------|------|--------|----------|------|
| POST | `/api/auth/login` | ❌ | ❌ | 登入 |
| POST | `/api/auth/forgot` | ❌ | ❌ | 忘記密碼（寄驗證碼） |
| POST | `/api/auth/reset` | ❌ | ❌ | 重設密碼（驗證碼） |
| GET | `/api/admin/projects` | ✅ | ❌ | 專案列表（含 token） |
| GET | `/api/admin/projects/:dev/:name` | ✅ | ❌ | 單一專案詳情 |
| POST | `/api/admin/projects` | ✅ | ❌ | 新增專案 |
| PUT | `/api/admin/projects/:dev/:name` | ✅ | ❌ | 更新專案 |
| DELETE | `/api/admin/projects/:dev/:name` | ✅ | ❌ | 刪除專案 |
| GET | `/api/admin/users` | ✅ | ✅ | 帳號列表 |
| POST | `/api/admin/users` | ✅ | ✅ | 新增帳號 |
| DELETE | `/api/admin/users/:username` | ✅ | ✅ | 刪除帳號 |
| PUT | `/api/admin/users/:username/password` | ✅ | ✅ | 重設密碼 |
| GET | `/api/admin/logs` | ✅ | ❌ | 操作紀錄（最近 100 筆） |
| DELETE | `/api/admin/logs` | ✅ | ✅ | 清除操作紀錄 |

## 初始帳號建立

部署時執行初始化腳本建立第一個 Admin 帳號：

```bash
node init-admin.mjs <帳號> <密碼> <email>
```

之後的帳號由 Admin 在管理頁面新增。

## 環境變數（`.env`）

| 變數 | 用途 | 備註 |
|------|------|------|
| `JWT_SECRET` | JWT 簽章密鑰 | 部署時設成隨機長字串 |

> SMTP 設定（`SMTP_HOST` / `SMTP_PORT` / `SMTP_USER` / `SMTP_PASS` / `SMTP_FROM`）為「忘記密碼」寄信功能所需，目前未啟用。Admin 可直接在管理頁面重設密碼，不需要 SMTP。未來需要時再設定 AWS SES。
